# سيرفر Push حقيقي — دوام المناوبين والبصمات

هذا السيرفر يرسل إشعارات حقيقية (Web Push) لجهازك بالضبط بالوقت المحدد، حتى لو
كان التطبيق مقفول تماماً من أيام. هذا هو الحل الوحيد الموثوق 100% للتنبيهات
بالخلفية على الويب.

## الخطوات

### 1) تثبيت المتطلبات محلياً (أو على السيرفر)
```bash
cd server
npm install
```

### 2) توليد مفاتيح VAPID (مرة واحدة فقط)
```bash
npm run generate-vapid
```
سيطبع لك مفتاحين:
```
VAPID_PUBLIC_KEY=...
VAPID_PRIVATE_KEY=...
```

### 3) إعداد ملف البيئة
انسخ `.env.example` إلى `.env` واملأ المفاتيح:
```bash
cp .env.example .env
```
ثم افتح `.env` وضع المفتاحين الناتجين من الخطوة السابقة.

### 4) تشغيل السيرفر محلياً للتجربة
```bash
npm start
```
لازم تشوف: `✓ سيرفر Push يعمل على المنفذ 3000`

### 5) رفع السيرفر لاستضافة فعلية (مطلوب HTTPS)
أسهل خيارات مجانية أو رخيصة:
- **Render.com** (مجاني للمشاريع الصغيرة) — ارفع مجلد `server` كـ Web Service،
  واضبط Environment Variables (VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_SUBJECT)
  من لوحة التحكم بدل ملف `.env`.
- **Railway.app**
- أي VPS عادي (DigitalOcean, Hetzner..) مع `pm2 start server.js`

بعد الرفع ستحصل على رابط مثل: `https://noc-push.onrender.com`

### 6) ربط التطبيق (index.html) بالسيرفر
افتح `index.html` وابحث عن:
```js
const PUSH_SERVER_URL = 'https://YOUR-PUSH-SERVER-URL';
const PUSH_VAPID_PUBLIC_KEY = 'YOUR-VAPID-PUBLIC-KEY';
```
استبدلهم بـ:
- رابط سيرفرك من الخطوة 5
- نفس `VAPID_PUBLIC_KEY` من ملف `.env`

### 7) رفع index.html + sw.js على استضافة بـ HTTPS
نفس الملاحظة السابقة: لازم يكونا على دومين حقيقي بـ HTTPS (PWA + Service Worker
+ Push كلهم يتطلبون HTTPS، ولا يعملون من فتح الملف مباشرة على الجهاز).

أمثلة استضافة ثابتة مجانية تناسب ملف HTML واحد:
- GitHub Pages
- Netlify / Vercel
- Cloudflare Pages

### 8) افتح التطبيق من الرابط الجديد على جوالك
- اسمح بالإشعارات لما يسألك المتصفح.
- بعدها التطبيق يسجل اشتراكك تلقائياً بالسيرفر.
- جرّب: فعّل أي منبه، أغلق المتصفح تماماً، وانتظر الوقت المحدد — لازم يوصلك
  إشعار حتى لو الجوال مقفول الشاشة.

## كيف يشتغل بالضبط؟
1. عند فتح التطبيق وإعطاء إذن الإشعارات، يشترك جهازك بخدمة Push من المتصفح
   (Google/Apple/Mozilla حسب المتصفح) ويرسل بيانات الاشتراك + قائمة منبهاتك
   لسيرفرك.
2. السيرفر يفحص كل 20 ثانية: هل وصل وقت أي منبه مفعّل عند أي مستخدم (بتوقيته
   المحلي الصحيح)؟
3. إذا وصل الوقت، يرسل السيرفر رسالة Push حقيقية عبر بنية تحتية المتصفح
   (هذا يعمل حتى لو التطبيق والمتصفح مقفولين تماماً، لأن نظام التشغيل نفسه
   يستقبل رسائل Push ويوقظ الـ Service Worker لحظياً).
4. الـ Service Worker (`sw.js`) يستقبل الرسالة ويعرض الإشعار فوراً.

## ملاحظات أمان
- ملف `subscriptions.json` يحتوي بيانات اشتراك المستخدمين — لا ترفعه لمستودع
  عام (git) بدون حماية.
- لمشروع شخصي/فريق صغير هذا التخزين بملف JSON كافٍ. لو عدد المستخدمين كبير،
  بدّل التخزين لقاعدة بيانات حقيقية (مثل SQLite أو MongoDB).
